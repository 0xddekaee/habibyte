FASE 1: CORE INFRASTRUCTURE (Sedang Berjalan - 95%)
--------------------------------------------------------------------------------

1. [‚úÖ] ARSITEKTUR CLEAN & MODULAR (Production Standard)
   Komentar Teknis:
   Kami menolak pendekatan monolitik yang kaku. Kode telah direfaktor total menjadi 6 modul terpisah (Crates) dalam satu Workspace yang kohesif.
   - `habibyte-p2p`: Modul networking mandiri (tidak tercampur dengan logika bisnis node).
   - `habibyte-ledger`: Logika blockchain murni yang terisolasi.
   - `habibyte-api`: Interface eksternal yang terpisah dari core logic.
   Dampak: Sistem menjadi sangat mudah dirawat (Maintainable), mudah diuji (Testable), dan siap untuk skala besar (Scalable).

2. [‚úÖ] SISTEM JARINGAN P2P TERDESENTRALISASI (Real Implementation)
   Komentar Teknis:
   Bukan sekadar simulasi. Kami telah mengimplementasikan stack `libp2p` standar industri.
   - Protokol Gossipsub: Memungkinkan penyebaran blok dan transaksi ke jutaan node secara efisien.
   - mDNS Discovery: Node secara cerdas menemukan peer lain dalam jaringan lokal tanpa konfigurasi manual.
   - Noise Protocol: Seluruh komunikasi antar-node dienkripsi, mencegah penyadapan (Eavesdropping).

3. [‚úÖ] REKAYASA PRIVASI IDENTITAS (Zero Data Leak)
   Komentar Teknis:
   Keamanan data warga adalah prioritas absolut.
   - Hashing SHA-256: NIK tidak pernah disimpan dalam bentuk teks asli (Plaintext).
   - Off-Chain Storage Strategy: Data sensitif medis/biometrik dienkripsi AES-GCM sebelum disimpan di IPFS/Storage, blockchain hanya menyimpan "kunci" atau hash-nya saja.

4. [‚úÖ] API GATEWAY STATEFUL
   Komentar Teknis:
   API Server telah terintegrasi langsung dengan Memori Ledger menggunakan `Arc<RwLock>`, memungkinkan pembacaan data secara real-time dan thread-safe (aman dari kondisi balapan/race condition).

5. [‚úÖ] STANDAR KUALITAS KODE (Code Hardening)
   Komentar Teknis:
   - Error Handling: Mengeliminasi penggunaan `unwrap()` yang berisiko *panic*, digantikan dengan penanganan error terstruktur.
   - Logging: Sistem pencatatan aktivitas node menggunakan Bahasa Indonesia baku yang informatif untuk memudahkan monitoring sistem.

6. [üîÑ] TESTING & DEVNET DEBUGGING (Quality Assurance)
   Komentar Teknis:
   Sebelum rilis kode sumber (Open Source), kami melakukan simulasi jaringan lokal (Local DevNet) multi-node untuk memastikan:
   - Node A dan Node B dapat saling menemukan (Peer Discovery).
   - API berjalan stabil tanpa memory leak.
   - Graceful Shutdown berfungsi sempurna untuk mencegah korupsi data.

RENCANA FASE SELANJUTNYA (FASE 2)
--------------------------------------------------------------------------------
Fokus: Persistensi Data & Perluasan Jaringan
Target: Q2 2026

1. [ ] INTEGRASI DATABASE PERSISTEN (RocksDB/Sled)
   Saat ini ledger masih berjalan di RAM (In-Memory). Langkah selanjutnya adalah menyimpan blok ke harddisk agar data tahan banting meskipun node dimatikan.

2. [ ] MEKANISME KONSENSUS LANJUTAN (Proof of Authority)
   Mengimplementasikan aturan validasi blok yang ketat, di mana hanya node validator terdaftar (Dukcapil/Pemerintah) yang berhak memvalidasi blok baru.

3. [ ] MANAJEMEN PEER GLOBAL (DHT Kademlia)
   Mengupgrade penemuan peer dari lokal (mDNS) menjadi global menggunakan Distributed Hash Table (DHT), memungkinkan node terhubung antar pulau/negara via internet publik.
